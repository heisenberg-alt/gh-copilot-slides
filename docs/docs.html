Documentation

Everything you need to install, configure, and use Slide Builder — whether through VS Code Copilot, the Go CLI, or Docker.

Installation

Python (MCP Server & Agents)

# Full install — all features
$ pip install -e ".[all]"

# Minimal — core features only
$ pip install -e "."

# PDF export (requires Chromium)
$ pip install -e ".[pdf]"
$ python -m playwright install chromium

# Research file support (PDF/DOCX reading)
$ pip install -e ".[research]"

Go CLI

$ go build -o bin/slides ./cmd/slides

CLI Reference

slides research

Full AI pipeline — research, curate, style, and export a presentation.

$ ./bin/slides research \
    --topic "AI in Healthcare" \
    --urls "https://example.com/report" \
    --slides 8 \
    --formats html,pptx,pdf \
    --pptx-template corporate.pptx

# Interactive mode (prompts for all options)
$ ./bin/slides research

slides new

Quick presentation with Copilot — no research pipeline.

$ ./bin/slides new \
    --topic "Startup Pitch" \
    --style neon_cyber \
    --slides 6 \
    --output pitch.html

slides edit

Edit an existing presentation with natural language instructions.

# Single edit
$ ./bin/slides edit --instruction "Change slide 3 to a quote slide"

# Interactive edit loop
$ ./bin/slides edit

# Change style
$ ./bin/slides edit --change-style

# Re-export to new formats
$ ./bin/slides edit --export pdf

slides convert

Convert a PowerPoint file to an HTML presentation.

$ ./bin/slides convert deck.pptx \
    --style bold_signal \
    --output web-deck.html

slides preview

Generate visual style previews filtered by mood.

$ ./bin/slides preview --mood calm --output ./previews/

slides list-styles

List all available style presets.

$ ./bin/slides list-styles

Docker

Multi-stage build: Go CLI compiled in Alpine, Python runtime on slim Debian. Runs as non-root.

$ docker build -t slide-builder .

# Run MCP server (default)
$ docker run -e GITHUB_TOKEN="$GITHUB_TOKEN" slide-builder

# Run Go CLI inside container
$ docker run --entrypoint slides slide-builder research --topic "Demo"

Agent Pipeline

The orchestrator coordinates five specialized agents in sequence:

StepAgentInputOutput
1ResearchTopic, URLs, filesSynthesized facts, stats, quotes, themes
2CuratorResearch bundle, slide countStructured slide array with types and narrative arc
3Style RecommenderContent tone, moodBest-fit style preset name
4ExporterSlides + styleHTML, PPTX, and/or PDF files
5EditorNatural language instructionModified slide array (loop)
Research Agent supports URLs (with SSRF protection), DuckDuckGo web search, and local files (.pdf, .docx, .pptx, .txt, .md) with a 50MB size limit.

Sessions

Presentations are automatically persisted as sessions in .slide-sessions/. Each session stores the slide data, style, research context, and edit history.

  • Iterative editing — run slides edit multiple times against the same deck
  • Style changes — switch presets without regenerating content
  • Re-export — generate additional formats from an existing session
  • History — full edit trail is preserved

Session IDs are hex-only (up to 32 characters) and files are written with restricted permissions (0600) for security.

PPTX Templates

Apply branding from an existing PowerPoint template. The Style Recommender extracts colors, fonts, and theme data, then applies them to the generated slides.

$ ./bin/slides research \
    --topic "Q4 Results" \
    --pptx-template corporate.pptx \
    --formats pptx
Template extraction works with standard PPTX files. Custom XML-heavy templates may lose some formatting details.

Output Features

Every generated HTML presentation includes:

  • Zero dependencies — single self-contained HTML file with inline CSS/JS
  • Keyboard navigation — arrow keys, space, home/end
  • Touch & swipe — mobile-friendly gesture support
  • Mouse wheel — smooth scroll-snap between slides
  • Progress bar — visual position indicator
  • Navigation dots — click to jump to any slide
  • Scroll-triggered animations — fade, scale, blur variants
  • Responsive design — viewport-fitting with clamp()
  • Reduced motion — respects prefers-reduced-motion media query
